# v0.0.4 建立自动化构建脚本和日志目录
**状态**：Planned  
**目标**：为长期开发准备可重复编译、打包和 QA 记录。

## 前置状态
- v0.0.3 已完成。

## 本次范围
- 创建 `BuildScripts/BuildEditor.bat`、`BuildScripts/PackageDev.bat`、`BuildScripts/CleanGenerated.bat` 占位。
- 创建 `Docs/Workflow/BuildGuide.md`，写明如何生成工程、编译、打 Development/Shipping。
- 创建 `Saved/AutomationReports` 忽略规则说明。
- 将常见打包地图写入 DefaultGame 或项目设置说明。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `BuildScripts/**`
- `.gitignore`
- `Docs/**`
- `Config packaging map entries only`

## Forbidden paths/actions
- gameplay code
- Content asset creation unrelated to build smoke
- destructive clean commands without path checks

## 跨版本关系
- Depends on: v0.0.3。
- Produces: 为长期开发准备可重复编译、打包和 QA 记录。
- Consumed by: v0.0.5。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- BuildEditor/PackageDevelopment/Automation entry points
- actual-result reporting

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Run automation/log checks if AutomationTestToolset is enabled; no gameplay asset writes.
- 写入前列出准确 Package/Tool/顺序/停止点；写入后分别报告保存、编译、重启加载和 PIE。

## Human Judgment / Credential
- Approve local UE installation path if auto-discovery finds multiple installations.
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- 执行 BuildEditor 或手动编译。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- 脚本文档存在。
- 至少能手动执行编译流程。
- 团队成员能照文档跑。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。

## 回滚条件
- Build/Editor/PIE 出现当前版本引入的阻断。
- 需要未批准的保护路径或破坏性资产操作。
- 当前实现要求提前创建未来系统。
- Blueprint/Save/Tag/资产路径出现未记录兼容破坏。

## 完成报告
按 AGENTS.md 的 11 项格式输出，并更新 DevelopmentLog、KnownIssues 和 ArchitectureDecisions。
