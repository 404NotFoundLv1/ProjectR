## 公共合同

### Ability 基础设施增量

在既有 `EPRAbilityActivationPolicy` 末尾追加：

- `ServerTriggered`

规则：

- 不重排已有枚举值；OnInputTriggered、WhileInputActive、Passive 的已有序列化值和行为不变。
- ServerTriggered 不得配置 InputTag；不会在授予、输入 Press 或 Passive 恢复时自动激活。
- 仅 Authority 可通过以下 C++ API 尝试激活：

```text
bool TryActivateAbilityByAbilityTag(
    FGameplayTag AbilityTag,
    FGameplayTagContainer& OutFailureTags)
```

- API 必须匹配唯一 AbilitySpec；非 Authority、空 Tag、零个或多个匹配均失败。
- 不接受类、对象路径、字符串、Blueprint 任意执行或 Debug 任意激活。
- Cooldown、Dead、Stunned 与 Commit 失败继续使用既有 failure/lifecycle 事件。

### Enemy 类型

`EPREnemyBrainState`：`Dormant`、`AcquireTarget`、`Pursue`、`Retreat`、`HoldRange`、`Attack`、`Staggered`、`Dead`。

`EPREnemyAttackPhase`：`None`、`Windup`、`Active`、`Recovery`、`Cooldown`。

`EPREnemyAttackKind`：`MeleeSweep`、`Projectile`。

`EPREnemySpawnStatus`：`Spawned`、`NotReady`、`NotAuthoritative`、`UnknownPrototype`、`InvalidTransform`、`SpawnFailed`、`Invalid`。

`FPREnemyAttributeDefaults` 必须只包含既有 11 项 AttributeSet 属性：Health、MaxHealth、Shield、MaxShield、Energy、MaxEnergy、AttackPower、MoveSpeed、CritChance、Permission、Resonance。

`FPREnemyPerceptionConfig` 固定包含 `SenseInterval`、`AcquireRange`、`LoseRange`、`PreferredMinRange`、`PreferredMaxRange`、`EdgeProbeForward`、`EdgeProbeDepth`。

`FPREnemyRuntimeState` 固定包含：

- `FGuid SpawnId`
- `FName CombatantId`
- `FGameplayTag PrototypeTag`
- `EPRAbilityTargetMobility Mobility`
- `EPREnemyBrainState BrainState`
- `EPREnemyAttackPhase AttackPhase`
- `FGameplayTag ActiveAttackTag`
- `TWeakObjectPtr<AActor> Target`
- `bool bAlive`
- `bool bGrounded`
- `double WorldTimeSeconds`

该运行时状态不复制、不存档，不 include HUD、QTE、Director、Reward 或 Boss 类型。

### Enemy Actor 的 GAS 生命周期

`APREnemyCharacter` 自身持有 `UPRAbilitySystemComponent` 与 `UPRAttributeSet`，并实现：

- `IAbilitySystemInterface`
- `IPRCombatantInterface`
- `IPRCombatFeedbackInterface`
- `IPRAbilityTargetInterface`

规则：

1. Enemy Actor 同时是 ASC Owner 与 Avatar；ASC 使用 `Minimal` replication mode，不改玩家 ASC 的 Mixed 模式。
2. Authority Spawn/BeginPlay 顺序固定为：验证 Prototype/SpawnId → `InitAbilityActorInfo(this, this)` → 一次性应用 `GE_Enemy_DefaultAttributes` → 添加 Alive/清除 Dead → 幂等授予 Enemy AbilitySet → 启动 StateTree。
3. 默认 GE、AbilitySet、Delegate 绑定和 StateTree 启动均必须幂等。任一步失败，Enemy 保持 Dormant，不能攻击。
4. `CombatantId` 固定为 `Enemy_<Prototype>_<32位SpawnGuid>`，在 Actor 生命周期内唯一稳定。
5. Dead 后取消攻击、停止 StateTree/移动、清理 Timer；不自动销毁、不发奖励。未来受控 Revive 只能恢复同一 Actor，不重复默认 GE/AbilitySet。

### 目标、移动与 StateTree

- Enemy AI 只获取本地单人 PlayerController 的当前 Pawn；目标必须有效、非 Dead、支持 AbilitySystem/Combatant，且 ASC 当前 Avatar 正是该 Pawn。
- 玩家 Pawn 替换时取消旧 Attack Token，释放旧弱引用，再重新解析当前 Avatar；不 include `APRPlayerCharacter`。
- `CanBeTargetedByAbility`：Alive 且初始化完成时接受现有六个 `Skill.*`；Dead、失效或未初始化时拒绝。
- Mobility 固定为：Melee/Range=Light，Shield=Heavy，Elite=Anchored。
- 共享 `/Game/ProjectR/Enemies/AI/ST_Enemy_Base` 只组合原生节点：`FPRSTTaskAcquireTarget`、`FPRSTTaskMoveToRangeBand`、`FPRSTTaskActivateAttack`、`FPRSTTaskWaitForReevaluate`、`FPRSTConditionEnemyState`。
- StateTree 由感知 Timer、ASC Tag、OnLanded、Ability lifecycle、目标失效和唯一 `Enemy.AI.Event.Reevaluate` 驱动；Enemy Actor/Brain 不启用永久 Tick。
- 出生时冻结 PlaneY；所有目标、方向、攻击和距离投影到 X/Z 平面，`Abs(CurrentY - PlaneY) <= 1cm`。
- 禁止 NavMesh、PathFollowing、AI MoveTo、EQS 和 3D 绕行。CharacterMovement 的引擎必需 Tick 可保留。
- 感知 Timer 固定 0.10s；Cooldown 期间按准确剩余时间重评，禁止快速重复激活。
- Pursue 只沿 X 移动至 PreferredMaxRange；Retreat 只沿 X 移动至 PreferredMinRange；Windup 开始后锁定攻击方向。
- 平台边缘前方 60cm、向下 120cm 探测，无地面即停止；不可达平台不跳跃或坠落追击，近战 HoldRange，远程仅在 LOS/合法距离内攻击。

### 攻击、CombatEvent、Guard/Dodge/AttackProxy

- 四种攻击均由原生 `UPREnemyAttackGameplayAbility` 执行；GA Blueprint 仅配置 CDO、AbilityTag、Cooldown GE，不能放业务 Graph。
- AbilitySpec SourceObject 必须是对应 Attack DataAsset。成功目标/范围/LOS 校验后 Commit，再进入 Windup；Commit 后取消不退还 Cooldown。
- 每次攻击使用 GUID Token；同一激活对同一目标最多 `ApplyDamage` 一次。
- 每次伤害必须构造完整 `FPRDamageRequest`：SourceId=AttackId，DamageSource=Attack DataAsset 或 Projectile，Instigator=Enemy，Target=当前玩家 Avatar，AbilityTag=`Enemy.Attack.*`，RawDamage 为表中公式，bCritical=false，DamageTags 含 PrototypeTag/AttackTag，ImpactOrigin 为接触点，IncomingDirection 为 X/Z 投影后 Enemy→Target 单位方向。
- `Applied`、`RejectedBlocked`、`RejectedInvulnerable`、`RejectedDead` 均结束本次命中；`Invalid` 取消攻击并记录日志；不得直接写 Health/Shield。
- Melee sweep/Projectile 先检查最近合法 `IPRCombatAttackProxyInterface`。代理成功消费后，本次攻击结束，不伤害玩家；代理自身负责既有 Outcome，Enemy 不重复广播。
- CounterProofWall 继续按 IncomingDirection 经 Combat mitigation 阻断。ShieldMinion/Elite 的护盾继续走现有 Shield spill-over，不建立 Guard HP。Shield>0 时可持有 `State.Guarding`；ShieldBreak 后移除。Elite 首个 ShieldBreak 应用 Stunned；既有 Damage Event 的 `Combat.Response.ShieldBroken` 是未来 QTE 的稳定事实。
- 本版本不实现全局 HitStop；必须验证粗帧和 Timer 延迟不会导致重复扣血、重复攻击或状态卡死。

## MCP Operation Manifest

1. 确认唯一 Editor、`PIE=false`、初始 Dirty=0、准确目标路径不存在且可编辑。
2. 重新审计官方 StateTree、Blueprint、GAS、DataAsset、Niagara、Audio、Programmatic Toolset，并记录能力、参数、返回值和日志。
3. Build 后加载新 C++ 类，按检查点 Manifest 创建资产；任何路径碰撞即停止。
4. 单事务配置并回读 GE SetByCaller/Duration/Tag、GA CDO、AbilitySet、Prototype/Attack DA、BP CDO、Registry、StateTree Schema/State/Task/Transition。
5. 任一类型、数组、引用、Tag、数值或 StateTree 节点不一致即回滚事务，不保存。
6. warnings-as-errors 编译 GA/GE/BP，运行 Data Validation；仅精确保存 Manifest，检查意外 Dirty=0。
7. 正常关闭重启并回读本检查点资产及冻结玩家/Combat 资产。
8. 加载但不保存 `L_CombatGym`，通过固定白名单 Spawn API 运行对应 PIE。
9. StopPIE 后检查 Actor、ASC、Spec、Effect、Timer、Delegate、Projectile、StateTree、Tag 和 Dirty，生成报告。

## 回滚、停止条件与完成门

以下任一情况必须停止并提出独立合同修订：需要替换 ASC/AttributeSet/Combat，必须改玩家技能识别 Enemy，ServerTriggered 不能保持旧枚举兼容，StateTree 必须依赖 NavMesh，攻击绕过 CombatSubsystem，存在重复初始化/攻击/事件/Timer，目标失效仍被攻击，白名单可被任意类或 Debug 绕过，或资产/重启/Dirty/历史回归失败。

每个检查点只反向撤销自身 C++、Config、文档和准确 Package 引用；Registry 先移除该检查点条目并精确保存。删除已保存 Package 必须逐项批准；禁止硬重置、Git clean、Save All、批量删除和 Fix Redirectors。

只有 A-E、全部 required 自动化、37 Package 精确保存、Dirty=0、重启、固定 PIE、人工手感、Future Compatibility Review、实施 ADR-022 和所需合同文档均实际 PASS，v0.2.1 才能在独立转换中标记 Completed。
