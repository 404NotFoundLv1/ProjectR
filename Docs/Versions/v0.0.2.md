# v0.0.2 接入基础模块依赖和 GameplayTags
**状态**：Planned  
**目标**：建立完整游戏所需的基础插件和标签体系。

## 前置状态
- v0.0.1 已完成。

## 本次范围
- Build.cs 添加 GAS、Tags、Tasks、EnhancedInput、UMG、AIModule、NavigationSystem、HTTP、Json、JsonUtilities、AudioMixer。
- DefaultGameplayTags.ini 添加输入、技能、状态、AI、QTE、法令、章节、房间、奖励标签。
- 创建 Tags 文档，说明每类 Tag 用途。
- 添加 `UPRTagLibrary` 统一获取常用 Tag。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `ProjectR.uproject`
- `Source/ProjectR/ProjectR.Build.cs`
- `Source/ProjectR/Core/PRTagLibrary.*`
- `Config/DefaultGameplayTags.ini`
- `Docs/**`

## Forbidden paths/actions
- GameplayAbility implementations
- GE/GA assets
- Content gameplay assets

## 跨版本关系
- Depends on: v0.0.1。
- Produces: 建立完整游戏所需的基础插件和标签体系。
- Consumed by: v0.0.3。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- GameplayTag taxonomy
- module dependency floor
- UPRTagLibrary access

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Inspect tags/toolsets and Project Settings after Build; asset writes are not required.
- 写入前列出准确 Package/Tool/顺序/停止点；写入后分别报告保存、编译、重启加载和 PIE。

## Human Judgment / Credential
- Restart Editor after plugin/module changes when Codex cannot safely restart a dirty Editor.
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- 编译。
- Project Settings 检查 GameplayTags。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- Tag 在编辑器可见。
- C++ 能 include 相关模块。
- TagLibrary 编译通过。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。

## 回滚条件
- Build/Editor/PIE 出现当前版本引入的阻断。
- 需要未批准的保护路径或破坏性资产操作。
- 当前实现要求提前创建未来系统。
- Blueprint/Save/Tag/资产路径出现未记录兼容破坏。

## 完成报告
按 AGENTS.md 的 11 项格式输出，并更新 DevelopmentLog、KnownIssues 和 ArchitectureDecisions。
