# v0.0.2 接入基础模块依赖和 GameplayTags
**状态**：Completed

**目标**：建立完整游戏所需的基础插件和标签体系。

## 前置状态
- v0.0.1 已完成。

## 本次范围
- Build.cs 添加 GAS、Tags、Tasks、EnhancedInput、UMG、AIModule、NavigationSystem、HTTP、Json、JsonUtilities、AudioMixer。
- DefaultGameplayTags.ini 添加输入、技能、状态、AI、QTE、法令、章节、房间、奖励标签。
- 创建 Tags 文档，说明每类 Tag 用途。
- 添加 `UPRTagLibrary` 统一获取常用 Tag。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `ProjectR.uproject`
- `Source/ProjectR/ProjectR.Build.cs`
- `Source/ProjectR/Core/PRTagLibrary.*`
- `Config/DefaultGameplayTags.ini`
- `Docs/**`

## Forbidden paths/actions
- GameplayAbility implementations
- GE/GA assets
- Content gameplay assets

## 跨版本关系
- Depends on: v0.0.1。
- Produces: 建立完整游戏所需的基础插件和标签体系。
- Consumed by: v0.0.3。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- GameplayTag taxonomy
- module dependency floor
- UPRTagLibrary access

- Inspect tags/toolsets and Project Settings after Build; asset writes are not required.
