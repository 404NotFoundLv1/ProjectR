# ProjectR v0.3.1 AI 实体、跟随、支援与压血不击杀

## 目标与完成边界

本版本一次性实现主同步 Companion 的唯一运行时 Pawn、稳定跟随、三种确定性支援、压血不击杀和供 v0.3.2 消费的只读 SupportEvent。它是单一完整版本，禁止拆分 A–E、子版本、检查点或中间功能提交。

唯一正式实施提交为：`v0.3.1 Add companion combat support no-kill rule`。

范围内：

- `UPRCompanionSubsystem` 继续是关系、Registry 和 run-local Primary 选择的唯一 owner。
- 新增 WorldSubsystem、Companion Pawn、Companion Component 和三个 Runtime DataAsset。
- 只生成一个 Primary Pawn；两个 Background 只有逻辑身份，不生成 Pawn、ASC、攻击 Timer 或战斗实体。
- Axiom 提供 Shield；Kindle 提供轻伤害；Null 提供控制/标记。
- 支援只经既有 Combat/GAS 窄入口，永不直接写 Health 或 Shield。
- 普通敌人保留至少 20% Health，Elite 至少 35%，Auditor 按既有阶段阈值停止直接伤害。
- 发布值类型 `FPRCompanionSupportEvent`，只供后续 QTE/Dialogue/Director 订阅。
- Codex 通过 MCP 创建、配置、编译、精确保存并回读全部三名 Companion Blueprint 与 Runtime 配置资产。

Non-goals：

- 不实现 QTE、对话、动态台词、濒死救援、关系 UI、Director、LLM、Save Schema 改动或新关系 API。
- 不修改玩家技能、Enemy/Boss 基础伤害、HUD、Input、地图、平衡值、GameplayTag、Config、Build.cs 或 uproject。
- 不创建 AIController、BehaviorTree、StateTree、NavMesh、PathFollowing 或第二套伤害/支援系统。

## 冻结上游合同

- v0.3.0 的三项身份 DataAsset、Registry、关系快照、Save Schema 2 和 `UPRCompanionSubsystem` API 不变。
- `UPRCombatSubsystem::ApplyDamage` 仍是唯一直接伤害入口；AttributeSet 仍是 Shield spill-over、Health Clamp 和死亡的唯一 owner。
- Enemy/Boss 只暴露稳定 Target、Combatant、Mobility、RuntimeState、Prototype 和 CombatEvent 合同；Companion 不 include 具体 Enemy/Boss Blueprint 类。
- `GE_State_Stunned` 保持既有 0.75s Duration 和 `State.Stunned`，本版本只读复用，不保存、不修改。
- v0.3.1 GameplayTag 增量为零：127 explicit、13 roots、61 checked getters 和 Config 语义均保持不变。

## 架构、公共 API 与生命周期

```text
UPRCompanionSubsystem (GameInstance: identity/relationship/sync)
                 ↓ OnPrimarySyncChanged
UPRCompanionRuntimeSubsystem (World: Pawn/support lifecycle)
                 ↓
APRCompanionPawn + UPRCompanionComponent (follow and support)
                 ↓
CombatSubsystem / existing GAS / AbilityTarget interfaces
```

新增 `UPRCompanionRuntimeSubsystem : UWorldSubsystem`。它只公开：

```text
FPRCompanionSupportEventNative& OnSupportEvent()
APRCompanionPawn* GetActiveCompanionPawn() const
```

新增的 `FPRCompanionSupportEvent` 必须只包含 EventId、CompanionId、SupportType、Result、SourceId、TargetId、RequestedMagnitude、AppliedMagnitude、HealthFloor、Overload、EffectiveIntervalSeconds、TargetLocation、WorldTimeSeconds。不得保存或包含 UObject、ASC、Spec/Effect Handle、Timer 或 Delegate。

`UPRCompanionRuntimeSubsystem` 在 World 初始化时读取当前 SyncState；有 Primary 时恰好 Spawn 一个 Pawn。Primary 变化、Profile Create/Load 清空、World Cleanup、EndPIE 和 Deinitialize 都必须以相同幂等顺序取消 Timer、解绑 Delegate、清空弱引用并销毁 Pawn。普通地图旅行不改变 SyncState；新 World 重新生成一个 Pawn。Player Pawn 替换由固定更新检测并重新绑定，旧引用不得保留。

`APRCompanionPawn` 不拥有 ASC、Health、Combatant 身份、AttackProxy、AIController 或持久化状态。它使用 Flying CharacterMovement；Actor 与 Companion Component 不开启永久 Tick，决策由 0.05s one-shot Timer 驱动。

## 跟随与碰撞合同

- 目标水平距离 180cm，允许 150–220cm，最大速度 550cm/s。
- Pawn 位于玩家 Z 上方 80cm；Y 锁定到玩家 Y，偏差不得超过 1cm。
- 玩家 X 速度决定后随侧；零速度沿用最后有效侧，初始为左侧。
- 超过 900cm 平面距离时允许一次传送恢复；必须先 Sweep/encroachment 验证 WorldStatic，失败后下次 Timer 重试。
- 正常移动使用 CharacterMovement 和 Sweep，不逐帧 `SetActorLocation`。
- Capsule：WorldStatic=Block；Pawn、WorldDynamic、Camera、Visibility=Ignore；`GenerateOverlapEvents=false`。Companion 不阻挡玩家、敌人或 Projectile。

## Runtime Data 与支援合同

新增 `UPRCompanionRuntimeDataAsset`，只允许固定字段：CompanionId、PawnClass、SupportType、BaseSupportInterval、SupportRange、SupportMagnitude、SupportEffectClass。RuntimeSubsystem 只加载本版本 Manifest 中的三条固定路径；不修改 Registry。

| Companion | SupportType | Base interval | Range | 正式行为 |
|---|---:|---:|---:|---|
| Axiom | Shield | 8.0s | self | 通过 `GE_Companion_Axiom_Shield` 增加 20 Shield |
| Kindle | LightDamage | 4.0s | 800cm | 通过 CombatSubsystem 造成 RawDamage=12，非暴击 |
| Null | ControlMark | 6.0s | 700cm | 普通/Elite 应用既有 Stunned；Boss 只发布标记事件 |

每次支援完成后重新读取 Primary 的 v0.3.0 关系快照：

```text
EffectiveInterval = BaseSupportInterval * (1 + Clamp(Overload, 0, 100) / 100)
```

Overload 0 使用基础间隔，100 使用两倍基础间隔。不修改关系、不刷新当前 Timer、不创建 Tag/GE。

目标选择仅在 Authority：已初始化、Alive、合法 AbilityTarget、范围内、Y 偏差不超过 1cm、Visibility/WorldStatic LOS 合法。按 X/Z 距离平方再按 TargetId 稳定排序，应用前再次验证。Null 优先未 Stunned 目标；没有合法目标时发布跳过事件。

## no-kill 合同

Kindle 对 Normal 的最低血线为 `0.20 * MaxHealth`，对 Elite 为 `0.35 * MaxHealth`。

```text
SafeRawDamage = CurrentShield + Max(0, CurrentHealth - HealthFloor)
AppliedRawDamage = Min(ConfiguredRawDamage, SafeRawDamage)
```

若结果不大于 epsilon，禁止调用 CombatSubsystem，发布 `SkippedAtHealthFloor`。支援伤害后必须验证目标仍不低于 HealthFloor；不能以直接属性写入修补结果。

Auditor：

- Sampling 只允许压到现有 `RuleAuditHealthRatio`。
- RuleAudit 只允许压到现有 `PredictionShieldHealthRatio`。
- PredictionShield、Dormant、Defeated 禁止 Companion 直接伤害。
- 两个比例必须从现有 Auditor DataAsset 读取，禁止在 Companion 代码重复硬编码。
- Axiom 的 Shield 始终只影响玩家；Null 对 Boss 只发布 `ControlMark` 事件而不应用 Stunned。
- 玩家仍可用正式 Combat 伤害补刀、推进阶段和击败 Boss。

## Future Compatibility Review

- v0.3.2 QTE subscribes only to `FPRCompanionSupportEvent` and uses existing controlled relationship Delta APIs; it cannot inspect private Pawn/Component state or bypass Combat.
- v0.3.3 Dialogue consumes persona, relationship and event facts only; it owns neither support targets nor timers.
- v0.3.4 rescue remains independent from Axiom Shield and cannot turn support into automatic revival.
- v0.4.0 Director may observe stable events but cannot directly spawn Companion Pawns or override their timers.
- Save stays Schema 2: no Pawn, target, timer, effect, handle or SupportEvent is persisted.
- Future Enemy/Boss types must expose stable Prototype/Mobility/phase contracts; Companion code does not branch on concrete Blueprint class.
