# v0.0.1 制定工程规范、命名规则和 Codex 协作规则
**状态**：Planned  
**目标**：避免后续多人/多月开发中结构失控。

## 前置状态
- v0.0.0 已完成。

## 本次范围
- 创建 `Docs/Workflow/CodingStandard.md`：类前缀、目录、注释、Blueprint 暴露规则。
- 创建 `Docs/Workflow/CodexWorkflow.md`：每次只执行一个小版本、修改文件列表、资产手动步骤、测试要求。
- 创建 `Docs/Workflow/AssetNaming.md`：BP_/WBP_/DA_/GE_/GA_/VFX_/SFX_ 命名。
- 新增 `UPRDeveloperSettings`，管理 Debug、MockDirector、Steam 功能开关。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `AGENTS.md`
- `CURRENT_VERSION.md`
- `Docs/**`
- `Source/ProjectR/Core/PRDeveloperSettings.h`
- `Source/ProjectR/Core/PRDeveloperSettings.cpp`

## Forbidden paths/actions
- Content/**
- future gameplay systems
- public asset renames

## 跨版本关系
- Depends on: v0.0.0。
- Produces: 避免后续多人/多月开发中结构失控。
- Consumed by: v0.0.2。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- APR/UPR/FPR/EPR naming
- completion report
- automation-first and MCP rules

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Read-only verification that DeveloperSettings becomes visible after Build/Editor reload.
- 写入前列出准确 Package/Tool/顺序/停止点；写入后分别报告保存、编译、重启加载和 PIE。

## Human Judgment / Credential
- Inspect the DeveloperSettings panel only if screenshot/Editor tool output is insufficient.
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- 编译。
- 检查 Project Settings。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- 文档可直接指导新成员。
- DeveloperSettings 可见且编译通过。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。

## 回滚条件
- Build/Editor/PIE 出现当前版本引入的阻断。
- 需要未批准的保护路径或破坏性资产操作。
- 当前实现要求提前创建未来系统。
- Blueprint/Save/Tag/资产路径出现未记录兼容破坏。

## 完成报告
按 AGENTS.md 的 11 项格式输出，并更新 DevelopmentLog、KnownIssues 和 ArchitectureDecisions。
