# v0.0.5 建立 Unreal MCP 安全资产生产基线
**状态**：Completed
**目标**：把已连接的 UE5.8 Unreal MCP 变成可审计、可回滚、可验证的正式开发能力。

## 前置状态
- v0.0.4 已完成。
- Unreal Editor 与 Codex 可通过本机 MCP 通信。

## 本次范围
- 记录当前 Toolset 能力矩阵和缺口。
- 选择性启用当前版本需要的官方专用 Toolset，并在重启后重新审计。
- 建立 MCP Operation Manifest、Package 碰撞检查、串行写入和精确保存流程。
- 在 /Game/ProjectR/MCPTest 创建 Blueprint、DataAsset 和模板地图复制的冒烟资产。
- 编译 Blueprint、验证引用、重启编辑器复查并执行 PIE 冒烟测试。
- 只有通用工具无法幂等覆盖的重复任务才建立 ProjectR Authoring Toolset。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `.codex/config.toml`
- `Docs/**`
- `Plugins/ProjectRAuthoringTools/** (only after gap proof)`
- `Content/ProjectR/MCPTest/**`
- `Content/__ExternalActors__/ProjectR/MCPTest/**`
- `Content/__ExternalObjects__/ProjectR/MCPTest/**`

## Forbidden paths/actions
- production asset overwrite
- asset delete/move/rename
- direct binary IO
- parallel Editor writes
- Save All；`AssetTools.save_assets` 不得传空数组
- Resave All

## 跨版本关系
- Depends on: v0.0.4。
- Produces: 把已连接的 UE5.8 Unreal MCP 变成可审计、可回滚、可验证的正式开发能力。
- Consumed by: v0.1.0。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- MCP Operation Manifest
- Package collision policy
- asset validation states
- Toolset capability matrix

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Create test DataAsset and Blueprint, add/configure component and graph, compile/save, use UE5.8 `WorldPartitionRenameDuplicateBuilder` to duplicate the test map, place Actor, restart reload, PIE.
- 确定性主 Package 为 `/Game/ProjectR/MCPTest/Blueprints/BP_MCPAuthoringSmokeActor`、`/Game/ProjectR/MCPTest/Data/DA_MCPAuthoringSmoke` 和 `/Game/ProjectR/MCPTest/Maps/L_MCPAuthoringSmoke`。DataAsset 的具体类型固定为非抽象 `/Script/Engine.PrimaryAssetLabel`；UE5.8 的 `/Script/Engine.PrimaryDataAsset` 为 Abstract，不得直接实例化。
- World Partition 写入仅允许 `/Game/__ExternalActors__/ProjectR/MCPTest/Maps/L_MCPAuthoringSmoke/**` 与 `/Game/__ExternalObjects__/ProjectR/MCPTest/Maps/L_MCPAuthoringSmoke/**`；UE 生成的哈希 Package 必须在 Builder 和 Actor 首次保存后分别枚举、冻结路径与 SHA-256，再允许下一阶段写入。
- 写入前列出准确 Package/Tool/顺序/停止点；保存只能使用 Operation Manifest 中的非空精确 Package 列表。Blueprint warnings-as-errors 编译通过、所有精确目标保存成功且全量意外 Dirty=0（Dirty Package 为 0）后，才允许重启并分别报告重载和 PIE。

## Human Judgment / Credential
- Codex 先精确保存 Operation Manifest 中的 Package、确认 Blueprint compile PASS 和 Dirty=0，再重启 Editor 并通过回读与截图检查测试资产；禁止 Save All。
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- MCP Blueprint/DataAsset 冒烟测试。
- Editor 重启后重新加载。
- 测试地图 PIE。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- Codex 能读取当前关卡并创建、配置、保存和重新加载测试 Blueprint/DataAsset。
- 所有写入都有 Operation Manifest，所有保存调用使用非空精确 Package 列表，且重启前和 PIE 后意外 Dirty Package 均为 0。
- 无直接二进制写入；测试资产范围限制在 `/Game/ProjectR/MCPTest/**` 及本任务批准的两个准确 External Package roots。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。
