# v0.1.1 GAS ASC、AttributeSet 与默认属性

**状态**：In Progress

**直接上游**：v0.1.0

**直接下游**：v0.1.2

**合同消费者**：v0.1.3、v0.2.3

## 目标

- 建立由 `APRPlayerState` 持有、`APRPlayerCharacter` 作为 Avatar 的稳定 GAS 生命周期。
- 建立可复制、可 Clamp、可由默认 GameplayEffect 初始化的战斗属性 Schema。
- 保证 Pawn 替换或重新占有时 ASC 和属性保留，默认属性不重复应用。
- 用统一 C++ 原生事件向未来 HUD 提交属性变化。
- 将 `MoveSpeed` 与 CharacterMovement 同步，同时保持 v0.1.0 的 `600 cm/s` 基线。
- 不提前实现伤害、死亡、技能、冷却、InputTag 路由或 HUD。

## 冻结合同与未来扩展点

- ASC 类型从本版本起冻结为 `UPRAbilitySystemComponent`。
- OwnerActor 永远为 `APRPlayerState`；AvatarActor 为当前 `APRPlayerCharacter`。
- AttributeSet 类型冻结为 `UPRAttributeSet`。
- 默认属性资产路径冻结为 `/Game/ProjectR/Effects/GE_DefaultAttributes`。
- 默认属性只由权威端对一个 PlayerState 成功应用一次；成功前的失败允许重试。
- Pawn 替换只能改变 Avatar，不重建 ASC/AttributeSet，也不重置当前属性。
- 未来 HUD 先读取 `GetAttributeSet()` 当前值，再订阅统一原生事件。
- v0.1.3 只能增量扩展 `UPRAbilitySystemComponent`，不能替换组件类型或重建输入生命周期。
- `Input.*` 继续表示稳定语义意图；本版本不把 InputAction 绑定到 ASC、Ability 类或 AbilitySpec。
- 不冻结 Blueprint callable 玩法 API；本版本只冻结 C++ 所有权、生命周期、属性和原生事件合同。
- 正式地图路径、OpenMap、Blueprint 父类、GameplayTags、v0.1.0 输入资产路径和 v0.0.5 MCP 安全合同不得改变。

## 公共 C++ 接口

### `PRAttributeTypes.h`

```text
FPRAttributeChange
- FGameplayAttribute Attribute
- float OldValue
- float NewValue

FPRAttributeChangedNative
- native multicast delegate
- payload: const FPRAttributeChange&
```

### `UPRAbilitySystemComponent`

```text
UPRAbilitySystemComponent : UAbilitySystemComponent
```

- 本版本不新增 Ability、Input、Cooldown、Cost 或 GameplayCue 业务方法。
- 类型存在的目的，是从第一版 GAS 基线开始冻结 Blueprint/序列化可见的 ASC 具体类型，避免 v0.1.3 替换基础组件。

### `UPRAttributeSet`

- 使用 `FGameplayAttributeData` 和项目统一的 `ATTRIBUTE_ACCESSORS_BASIC` 宏。
- 11 项属性全部使用 `ReplicatedUsing` 指向各自的 `OnRep_*`。
- 实现 `PreAttributeChange`、`PostGameplayEffectExecute` 和 `GetLifetimeReplicatedProps`。
- 每个 RepNotify 使用 `GAMEPLAYATTRIBUTE_REPNOTIFY`。
- 复制注册使用 `REPNOTIFY_Always`，以便未来预测修正和客户端事件保持一致。
- 不实现伤害、死亡、Shield spill-over、资源消费或 CombatEvent。

### `APRPlayerState`

```text
UAbilitySystemComponent* GetAbilitySystemComponent() const override
UPRAbilitySystemComponent* GetProjectRAbilitySystemComponent() const
const UPRAttributeSet* GetAttributeSet() const
bool InitializeAbilitySystemForAvatar(APRPlayerCharacter* Avatar)
void ClearAbilityAvatar(APRPlayerCharacter* Avatar)
FPRAttributeChangedNative& OnAttributeChanged()
```

- 实现 `IAbilitySystemInterface`。
- 构造时创建 ASC 和 AttributeSet 默认子对象。
- ASC 开启复制，ReplicationMode 固定为 Mixed。
- AttributeSet 通过 `AddAttributeSetSubobject` 注册。
- `NetUpdateFrequency = 100.0f`。
- `DefaultAttributesEffect` 为 `EditDefaultsOnly` 的 `TSubclassOf<UGameplayEffect>`。
- 统一事件只在 `OldValue` 与 `NewValue` 实际不同时广播。
- 11 项标准 GAS 属性变化 Delegate 的注册与解绑必须幂等。
- 默认 GE 的“一次成功”状态归属于 PlayerState，不归属于 Pawn。

### `APRPlayerCharacter`

- 实现 `IAbilitySystemInterface`。
- 覆写 `PossessedBy`、`OnRep_PlayerState`、`UnPossessed` 和 `EndPlay`。
- `GetAbilitySystemComponent()` 转发到当前 `APRPlayerState`。
- 初始化后绑定 `MoveSpeed` 的标准 GAS 属性变化 Delegate，并立即同步当前值。
- `UnPossessed`/`EndPlay` 解除旧的 MoveSpeed Delegate 绑定。
- 只有 ASC 当前 Avatar 确实为该 Character 时才允许清除 ActorInfo，避免旧 Pawn 清除新 Pawn 的 ActorInfo。
- 不改变 v0.1.0 的输入绑定、平面移动、跳跃、空中反向、Mesh 转向或侧视相机接口。

## 属性 Schema

| 属性 | 默认值 | 有效范围 | 语义 |
|---|---:|---:|---|
| MaxHealth | 100 | 1–100000 | Health 上限 |
| Health | 100 | 0–MaxHealth | 当前生命；本版本不触发死亡 |
| MaxShield | 50 | 0–100000 | Shield 上限 |
| Shield | 50 | 0–MaxShield | 当前护盾；本版本不承担伤害 |
| MaxEnergy | 100 | 0–100000 | Energy 上限 |
| Energy | 100 | 0–MaxEnergy | 未来技能资源 |
| AttackPower | 10 | 0–100000 | 未来伤害公式的绝对攻击力 |
| MoveSpeed | 600 | 0–2000 | CharacterMovement MaxWalkSpeed，单位 cm/s |
| CritChance | 0.05 | 0–1 | 暴击概率；原路线 `Crit` 的明确映射 |
| Permission | 0 | 0–999999 | 本轮权限货币 |
| Resonance | 0 | 0–100 | 本轮共鸣计量 |

### Clamp 规则

- Health、Shield、Energy 始终 Clamp 到对应 Max。
- Max 属性变化时，旧 Max 大于零则保持当前值百分比；旧 Max 为零则仅 Clamp，不自动补满。
- Max 降低后当前值不得超过新 Max。
- AttackPower、MoveSpeed、CritChance、Permission、Resonance 使用表中固定安全范围。
- 初始化 GE 的 Override 值也必须经过相同合法性检查，不能绕过 AttributeSet Clamp。
- Clamp 只保证数值合法，不广播 CombatEvent，不触发伤害、死亡、恢复、资源消费或其他战斗语义。

## 默认属性 GameplayEffect

### Package

```text
/Game/ProjectR/Effects/GE_DefaultAttributes
```

### 配置

- Parent：`/Script/GameplayAbilities.GameplayEffect`
- DurationPolicy：Instant
- Modifier operation：Override
- Modifier 顺序必须精确为：
  1. MaxHealth = 100
  2. MaxShield = 50
  3. MaxEnergy = 100
  4. Health = 100
  5. Shield = 50
  6. Energy = 100
  7. AttackPower = 10
  8. MoveSpeed = 600
  9. CritChance = 0.05
  10. Permission = 0
  11. Resonance = 0
- 不配置 Period、Executions、Granted Tags、Application Requirements、Stacking 或 GameplayCue。

### 应用规则

- `InitAbilityActorInfo(PlayerState, Character)` 成功后才尝试应用。
- 仅 Authority 应用。
- 每个 PlayerState 只成功应用一次；成功前的缺失资产、Spec 创建失败或应用失败可以重试。
- 重复 `PossessedBy`、`OnRep_PlayerState` 或相同 Avatar 初始化不得重复应用。
- Pawn 替换只重新初始化 Avatar 并同步 MoveSpeed，不重新应用默认 GE。
- 客户端不自行应用 GE，只接收复制属性。
- 默认 GE 为 Instant，运行时检查时 ActiveEffects 应为空。

## 生命周期

1. PlayerState 构造 ASC 和 AttributeSet 默认子对象。
2. PlayerState `BeginPlay` 幂等注册 11 项属性变化 Delegate。
3. 服务端 Character `PossessedBy` 调用 `InitializeAbilitySystemForAvatar`。
4. 客户端 Character `OnRep_PlayerState` 调用同一入口。
5. PlayerState 初始化 ActorInfo，并在 Authority 上应用默认 GE 一次。
6. Character 绑定 MoveSpeed Delegate，将 MaxWalkSpeed 立即同步为 600。
7. 同一 Pawn 重复初始化不重复绑定，也不重复应用默认 GE。
8. 新 Pawn 复用同一 PlayerState 时成为新 Avatar，并继承当前属性。
9. 旧 Pawn 只解除自己的 MoveSpeed 绑定；若 ASC Avatar 已变为新 Pawn，不得清除新 ActorInfo。
10. PlayerState `EndPlay` 解除统一属性 Delegate。

## MCP Operation Manifest

所有 Editor 写操作必须对同一个 UnrealEditor 实例串行执行：

1. 检查 Editor 身份、`PIE=false`、目标 Package 碰撞、可编辑性和当前 Dirty 状态。
2. 启用 `GASToolsets`、完成 BuildEditor 并正常重启后，审计实际新增 Toolset/Tool；静态预期为 AttributeSetToolset、AbilitySystemInspectorToolset、GameplayCueToolset，共 14 个工具。本版本只调用 AttributeSet 与 AbilitySystemInspector 的只读能力。
3. 使用 AttributeSetToolset 回读 11 项 Native Attribute。
4. 使用 BlueprintTools 创建 `GE_DefaultAttributes`。
5. 使用 ObjectTools 配置 DurationPolicy 和 11 项 Modifier。
6. 完整回读 GE CDO；任一顺序、Attribute、Operation 或值不匹配即停止，不保存。
7. 设置 `BP_PlayerState` CDO 的 `DefaultAttributesEffect`。
8. 以 warnings-as-errors 编译 GE、`BP_PlayerState`、`BP_PlayerCharacter` 和 `BP_PlayerController`。
9. 只保存非空、准确 Manifest 中的 Package。
10. 对全量可查询资产执行 Dirty 门，要求意外 Dirty=0。
11. 正常重启 Editor，重新加载并回读：GE 类、Duration、11 项 Modifier；`BP_PlayerState` 父类和 GE 引用；`BP_PlayerCharacter`/`BP_PlayerController` 父类与 v0.1.0 配置。
12. 打开 `L_CombatGym` 运行 PIE：AbilitySystemInspector 分别检查 PlayerState 和 Character 返回同一 ASC 数据；11 项 Base/Current 值精确匹配；ActiveEffects 为空；GrantedAbilities 为空；ActiveTags 未增加；运行既有 `RunPIEInputSmoke`，不得修改该工具。
13. StopPIE 后再次验证 Dirty=0、日志和截图。
14. 使用 `BuildScripts/AutomationReport.bat` 生成统一报告。

如果官方 BlueprintTools/ObjectTools 不能准确配置并回读 GE 嵌套 Modifier：

- 保存工具名、参数、返回值和相关 Editor 日志。
- 不进行猜测性保存，不把机械操作转交用户。
