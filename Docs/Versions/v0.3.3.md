# 目标

在单一完整版本中交付本地确定性的 Companion 战斗短句与安全状态有限选择：当前 Primary Companion 是唯一发言者；同一事件由 Axiom、Kindle、Null 的不同数据驱动文本表达；选择通过稳定 ID 和既有 `FPRRelationshipDelta` 影响关系。

# 范围与冻结边界

- 只消费 `FPRCombatEvent`、`FPRQTEResult`、`FPRCompanionSupportEvent`、`FPRBossRuntimeState`、`FPRPrototypeRunResult` 及 `FPREnemyRuntimeState` 的稳定值事实；不得读取或控制其私有 RuntimeState、Timer、Widget、Target 或 UObject。
- 不实现 LLM、HTTP、网络、自由文本、TTS、长期记忆、Room、Director、画像、濒死救援、自动复活、牺牲选择、伤害或支援改写。
- 不修改 Combat、QTE、Companion AI、Boss、CombatHUD、Save Schema、GameplayTag、Input、Config、地图、Ability、Enemy 或 AttributeSet。
- UI 只显示状态并提交有限 `ChoiceId`；不拥有伤害、关系、队列、保存或安全状态权威。

# 上下游与稳定合同

- 上游：v0.3.0 Companion/Relationship、v0.3.1 SupportEvent、v0.3.2 QTEResult。
- 下游：v0.3.4 仅消费值型 Dialogue ChoiceResult；v0.4.2 可增加正式 Room-safe 值事件；v0.5.2 可在同一校验入口前接入离线/安全 LLM Provider。
- Relationship 继续只由 `UPRCompanionSubsystem::ApplyRelationshipDelta` 更新；成功变化后仅调用一次 `UPRSaveSubsystem::RequestSaveCurrentProfile`。不保存文本历史、选择队列、Widget、Timer、Delegate、UObject 或运行时 Handle。

# Dialogue 数据与状态机

新增 `EPRDialogueTriggerKind`：`CriticalLowHealth`、`LowHealth`、`QTESuccess`、`QTEFailure`、`SupportApplied`、`SupportRejected`、`BossPhaseChanged`、`PredictionBlocked`、`SafeCombatCleared`。

新增值型 `FPRDialogueRuntimeState`、`FPRDialogueChoiceOption`、`FPRDialogueChoiceResult`，以及唯一世界所有者 `UPRCompanionDialogueSubsystem`：

```text
Idle -> Bark -> Idle
Idle -> Choice -> Resolved / Expired / Cancelled -> Idle
```

- 同时只有一个 Active 项，队列最多 4 项；排序为 Priority 降序、事件时间升序、LineId 字典序。
- 最近 64 个 EventId 在 10 秒内去重；全局最短间隔 1.25 秒、10 秒最多 3 条短句；Bark 显示 2.5 秒、排队项 3 秒过期。
- 冷却：Critical 12 秒、QTE/Boss 8 秒、Low/Support 10 秒、Safe 30 秒。新项至少高当前 20 Priority 且当前已显示 0.35 秒才可抢占。
- CriticalLowHealth 只由已有 `FPRCombatEvent` 推导：`0 < RemainingHealth / MaxHealth <= 0.15`；LowHealth 为 `(0.15, 0.35]`，不触发救援。
- 安全选择仅在观察到正式敌人后全部死亡或 Boss 完成、并连续 2.0 秒无新伤害/新敌人时出现。2.0 秒大于现有 QTE 最大 1.8 秒窗口，因此不读取 QTE 私有状态。
- 战斗重启、玩家死亡、Pawn replacement、旅行、World Cleanup、PIE Stop 和 Deinitialize 均取消 Choice，不应用 Delta，并清理 Widget、Timer、Delegate、队列和本地组件。

# 固定文本与选择

每个 Companion DataAsset 必须包含下列相同 LineId，文本不同：

| Trigger / Priority | Axiom | Kindle | Null |
|---|---|---|---|
| CriticalLowHealth / 100 | 生命曲线临界，立即脱离。 | 别硬撑，先拉开！ | 再挨一下，我就替你写遗言。 |
| QTESuccess / 90 | 协作窗口闭合，结果符合预期。 | 漂亮！就是这个节奏！ | 行吧，这次配合得像回事。 |
| QTEFailure / 90 | 协作窗口失效，重置节奏。 | 没接上？下一次抢回来！ | 错过了，别假装那是战术。 |
| BossPhaseChanged / 90 | 审计阶段变化，重新评估攻击序列。 | 它换阶段了，继续压上！ | 看吧，它开始改规则了。 |
| PredictionBlocked / 90 | 当前模式已被预测，切换技能。 | 它读懂你了，换招砸开！ | 重复答案被判错，换一道题。 |
| LowHealth / 60 | 生命资源偏低，降低交换频率。 | 血量不妙，别贪刀！ | 你的血条比计划还短。 |
| SupportRejected / 60 | 支援条件不成立。 | 没接上，重来！ | 连系统都不配合你。 |
| SupportApplied / 50 | 支援已生效。 | 接住，继续上！ | 我只是顺手。 |
| SafeCombatCleared / 20 | 区域稳定，开始复盘。 | 清完了，来复盘！ | 终于安静，勉强聊两句。 |

安全选择最多两个，且仅在已加载 Profile 时显示：

- `Choice.Acknowledge`：Trust +1、Affection +1；文本分别为“认可这次协作”“夸夸这次配合”“承认你有点用”。
- `Choice.Analyze`：Trust +1、Evaluation +1；文本分别为“分析下一次风险”“说说哪里还能更猛”“挑出这次的问题”。

Choice 15 秒过期。非法、重复、取消或无 Profile 的选择均不产生关系变化或保存请求。

# MCP Operation Manifest

1. 确认唯一 Editor、正确项目、PIE=false、Dirty=0；审计官方及 ProjectR Toolset Schema。
2. BuildEditor 后重启；确认五个 Create 路径不存在且可编辑。
3. 使用官方能力或固定 Authoring Tool 串行创建并配置五项资产。
4. 回读三份九项 Line、Choice Delta、Registry 顺序与 Widget Class；不匹配即停止且不保存。
5. warnings-as-errors 编译 WBP，运行 Data Validation。
6. 只精确保存五项新 Package；Manifest 外 Dirty 立即停止。
7. 正常重启并重新回读；确认上游 Package 与两张 Gym 地图未变化。
8. 运行无参数 `RunPIECompanionDialogueSmoke()` 和 `RunPIECompanionDialogueHumanPreview()`；StopPIE 后确认 Widget、Timer、Delegate、队列、Actor、测试槽和 Dirty 均清理。

禁止 Save All、Resave All、Fix Redirectors、地图保存、资产删除/移动/改名/复制，以及普通文件 I/O 修改 `.uasset/.umap`。
