## 冻结公共合同

### Boss 运行时

`EPRAuditorBossPhase` 固定为 `Dormant`、`Sampling`、`RuleAudit`、`PredictionShield`、`Defeated`；只能单向推进。

`FPRBossRuntimeState` 固定包含 `BossId`、`SpawnId`、`Phase`、`Health`、`MaxHealth`、`Shield`、`MaxShield`、`ActiveRuleId`、`PredictedAbilityTag`、`ActiveAttackTag`、`AttackPhase`、`PhaseSequence`、`bDefeated`、`WorldTimeSeconds`。

`FPRPrototypeRunResult` 固定包含 `CompletionId`、`BossId`、`BossSpawnId`、`CounterproofFragmentsAwarded`、`WorldTimeSeconds`；碎片数永远为 1。

`UPRBossSubsystem : UWorldSubsystem` 至少提供：

```text
bool GetActiveBossState(FPRBossRuntimeState& OutState) const
FPRBossStateChangedNative& OnBossStateChanged()
FPRPrototypeRunCompletedNative& OnPrototypeRunCompleted()
```

RuntimeState/Result 只包含值类型、稳定 ID 与 GameplayTag；不含 UObject、ASC、Spec、EffectHandle、Timer 或 Delegate，不复制、不存档。Defeated 与 PrototypeRunCompleted 各只能发布一次。

`APRBossAuditor : APREnemyCharacter, IPRCombatMitigationInterface` 持有 `UPRAuditorBossComponent`；它只将现有 Combat mitigation 查询转发给组件。`APREnemyCharacter` 不得出现 Auditor 类型分支。

### Enemy 选择、Registry 与地图

- 新增窄 `IPREnemyAttackSelectionInterface`。Enemy Brain 有选择器时只接受其返回的、属于当前 Prototype 已验证 AttackDefinitions 的 AttackTag；无选择器严格维持当前第一个配置 Attack 行为。Auditor component 是唯一 v0.2.2 实现者。
- `UPREnemySubsystem` 去除四 Tag 硬编码：Registry 内容成为唯一固定白名单；四个既有 Entry 的顺序/语义保持不变，`Enemy.Type.AuditorBoss` 追加第五项。Spawn 继续只接受已加载 Registry Entry，拒绝任意类、对象、路径、字符串及 Registry 外 Tag。
- `APRBossPrototypeGameMode : APRNetworkRunGameMode` 仅在 Authority、Registry ready 后以固定 `Enemy.Type.AuditorBoss` 生成；失败记录后停止，不猜测类/路径/重试。
- `L_BossGym` 只覆盖 `BP_BossPrototypeGameMode`，验证既有 Player/Camera/Game Framework，不扩建关卡美术。

### P1、P2、P3 与 Defeat

- P0：Health/MaxHealth=900/900，初始 Shield/MaxShield=120/120，AttackPower=18，MoveSpeed=320，Acquire/Lose=1600/1900，Preferred=180–650，P2/P3=66%/33%，PredictionShield=240。
- P1（HealthRatio > 0.66）：使用 Sweep、LockShot、普通 Shield；订阅既有 CombatEvent，仅记录最近 12 秒、最多 16 条合法玩家 `Skill.*`、距离、PerfectTiming/Dodge outcome 与时间；不 include 六个具体技能类，不保存或联网。
- P2（<=0.66）：只使用 `Rule.DistanceCorrection` 或 `Rule.RepetitionPenalty`。同一 Skill 在窗口内至少三次时选 RepetitionPenalty，同频按 Tag 字符串字典序；否则至少三样本且中位距离 >650 时选 DistanceCorrection；数据不足固定回退 DistanceCorrection。DistanceCorrection 只提高 LockShot 选择优先级。被采样重复 Skill 在 P2 再次出现时武装一次 Counter，Brain 将其目标距离改为 0–300，成功选择后清除武装。
- P3（<=0.33）：选次数最多的合法 Skill，同频按 Tag 字符串字典序。存在样本时应用 Infinite `GE_Boss_Auditor_PredictionShield`，把 MaxShield 与 Shield 设为 240。预测 Skill 命中时 mitigation 返回 `Blocked`，唯一 ResponseTag 是 `Combat.Response.PredictionBlocked`，既有 CombatSubsystem 发布一次 `Combat.Event.DamageRejected`；Blocked 不改属性或 GE。其他 Skill 正常伤害 Shield；Shield 为 0、死亡或 EndPlay 时移除 GE；无样本不设预测 Tag 且不阻断。
- Defeat：停止 Brain、Attack、Projectile、Timer、Delegate 与表现，发布一次 Defeated 和一次 Completion Result；只显示运行时 1 个反证碎片，不创建掉落、背包、Save 或账号删除。
