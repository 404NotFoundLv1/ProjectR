# v0.0.0 创建 Steam 版 UE 工程与仓库
**状态**：Completed

**目标**：把已经存在的 UE5.8 模板工程转化为可回滚、可审计的正式开发基线。

## 前置状态
- ProjectR UE5.8 C++ 工程已存在并可由 Unreal Editor 打开。
- 工程尚未初始化 Git，也没有正式基线文档。

## 本次范围
- 盘点 ProjectR.uproject、Source、Content、Config 和模板 Variant。
- 初始化 Git 与 UE .gitignore，建立首次可回滚提交边界。
- 记录引擎路径、工程关联、首次 Build、Editor 打开和 PIE 结果。
- 建立 Docs、BuildScripts、DevelopmentLog、KnownIssues 和 ArchitectureDecisions。
- 模板 Variant 暂时保留为参考，禁止正式系统新增依赖。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `.gitignore`
- `Docs/**`
- `BuildScripts/**`
- `ProjectR.uproject (read-only)`
- `Source/** (read-only inventory)`
- `Content/** (read-only inventory)`
- `Config/** (read-only inventory)`

## Forbidden paths/actions
- Source/** gameplay edits
- Content/** asset edits
- Config/** edits
- Unreal MCP writes

## 跨版本关系
- Depends on: UE5.8 模板工程与已连接 MCP。
- Produces: 把已经存在的 UE5.8 模板工程转化为可回滚、可审计的正式开发基线。
- Consumed by: v0.0.1。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- Project/module name ProjectR
- source/content/config inventory
- build and PIE baseline record

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Read current level, selected actors/assets, logs and toolsets only; no writes.
- 写入前列出准确 Package/Tool/顺序/停止点；写入后分别报告保存、编译、重启加载和 PIE。

## Human Judgment / Credential
- Confirm first PIE movement/visual result only if Codex cannot perform subjective validation.
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- C++ 编译。
- 默认 PIE。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- 项目可编译可打开。
- Git 不包含 Binaries/Intermediate/Saved。
- 工程名和模块名稳定。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。

## 回滚条件
- Build/Editor/PIE 出现当前版本引入的阻断。
- 需要未批准的保护路径或破坏性资产操作。
- 当前实现要求提前创建未来系统。
- Blueprint/Save/Tag/资产路径出现未记录兼容破坏。

## 完成报告
按 AGENTS.md 的 11 项格式输出，并更新 DevelopmentLog、KnownIssues 和 ArchitectureDecisions。
