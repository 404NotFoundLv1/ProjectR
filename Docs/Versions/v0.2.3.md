# ProjectR v0.2.3 战斗 HUD、技能冷却与事件反馈

- 目标：交付只读、事件驱动的正式战斗 HUD，在 CombatGym 与 BossGym 自动挂载并清晰展示玩家资源、六项正式技能运行态、事件提示条和既有 Boss 战状态。

## 冻结 HUD 合同

```text
BP_NetworkRunGameMode / BP_BossPrototypeGameMode
                    ↓ HUDClass
              BP_CombatHUD
                    ↓
              APRCombatHUD
                    ↓
           WBP_CombatHUD
          ├─ WBP_PlayerResources
          ├─ WBP_SkillBar
          │   └─ WBP_SkillSlot × 6
          ├─ WBP_CombatFeedback
          ├─ WBP_BossHealth
          └─ WBP_BossPrototypeResult
```

- `APRCombatHUD : AHUD` 仅为本地 `APRPlayerController` 创建一次 `WBP_CombatHUD`；`EndPlay`/Widget destruct 必须解绑每一项 Delegate、清除每一个 Timer，并且不得在服务器创建 UI。
- HUD 只订阅 AttributeChange、AbilityLifecycleEvent、CombatEvent 和 Boss Runtime/Completion；不得调用伤害、保存、生成、奖励、Director 或任何权威玩法 API。
- 资源区显示 Health/MaxHealth、Shield/MaxShield、Energy/MaxEnergy、Permission、Resonance。初始快照取 AttributeSet，之后只按属性事件刷新。
- 技能栏固定按 `DA_DefaultAbilitySet` 已授予顺序展示：ShadowThrust、FireSlash、ThunderDrop、AfterimageDodge、VectorHook、CounterProofWall。展示文本由稳定 `Skill.*` Tag 映射；不修改六份 Skill DataAsset。每格展示可用、Active、Held、EnergyCost、CooldownDuration、CooldownRemaining 与最新失败原因。
- 冷却仅在剩余时间大于零时用 0.1 秒 UI Timer 查询既有 RuntimeState；归零后立即停止。不存在 HUD Tick 或全量轮询。
- Boss 区直接复用 `UPRBossHealthWidget`、`UPRBossPrototypeResultWidget`：显示 Health/Shield、Phase、Attack Windup、PredictionShield、PredictedAbilityTag、完成结果。`ActiveRuleId` 必须呈现为“审计者本地规则”，绝不称为 Director 法令。

### 事件提示条

- `WBP_CombatFeedback` 位于右侧，最多显示最新三条，每条默认 2.5 秒后由自己的 Timer 移除。
- 每个 `FPRCombatEvent.EventId` 只消费一次。同一事件可在同一条内表达普通/暴击命中、护盾吸收、破盾、生命伤害或预测阻断，避免同一命中重复刷屏。
- 同一语义事件在 0.2 秒内具有相同提示类型、来源、目标和 AbilityTag 时合并，显示次数和累计数值；死亡、Ability ActivationFailed 和 CommitFailed 不合并。
- 固定提示语义：普通命中、暴击、护盾吸收、破盾、玩家受击、玩家护盾受击、敌人死亡、玩家死亡、`Combat.Response.PredictionBlocked`，以及 Ability 激活/Commit 失败。
- 不实现世界飘字、DamageNumber、音频、VFX、Niagara 或新玩法事件；提示条只是现有事件的只读消费者。
