# 目标

在一个完整版本中交付每轮一次的濒死分歧：主同步 Companion 在玩家正式死亡后提供“挽救、直接离开、直面挑战”三项有限选择。系统以本地确定性方式复用既有 Combat、Relationship、Dialogue 与 Save 合同，并发布供后续版本消费的值型结果。

# 范围与冻结边界

- 只在玩家正式 `Combat.Event.Death` 后处理；伤害链、死亡顺序和复活继续唯一经 `UPRCombatSubsystem`。
- 资格必须同时满足：Authority、当前 Player Avatar 已 Dead、有效 Primary、已加载 Profile、`Trust >= 50`、`Overload < 80`、本轮尚未消费。
- P0 的“每轮”固定为当前 GameInstance/Profile 运行会话：普通地图旅行不重置；成功 Create/Load Profile 才开始新会话。v0.4.0 的正式 RunState 只替换此重置信号，不改变结果 Schema。
- 不实现 PlayerProfile、Director、Room、撤离、资源保留、奖励、账号删除、墓园、技能树、网络、LLM、TTS、自由文本或自动复活。
- 不修改 GameplayTag、Input、Config、Save Schema、地图、Combat/QTE/Companion/Boss/CombatHUD 核心逻辑；不把 ReverseBurn QTE 或 Companion Support 升级为 Revive。
- RuntimeState、Widget、Timer、Delegate、Actor、ASC、Effect/Spec Handle 和弱 UObject 引用不得进入 Save。

# 结果、状态机与选择

新增 `EPRDivergenceState`（Idle、AwaitingChoice、Resolving、Completed）、`EPRDivergenceChoice`（None、Rescue、Leave、FaceChallenge）、`EPRDivergenceResolution`（Applied、RejectedIneligible、RejectedAlreadyConsumed、RejectedCombat、RejectedInvalid、Expired、Cancelled）与 `EPRDivergenceFutureDisposition`（None、RescueEvacuationRequested、LeaveRunRequested、ChallengeContinues）。

新增值型 `FPRDivergenceRuntimeState`、`FPRDivergenceResult` 和 Result/State Native Delegate。结果必须含 Request/Result/DeathEvent ID、Companion、Choice、Resolution、FutureDisposition、Revive 状态与比例、关系前后快照、Delta、SaveRequestId、最近有效同 Primary Dialogue Result/Choice ID 和 WorldTime；不含 UObject。

```text
Idle -> AwaitingChoice -> Resolving -> Completed
AwaitingChoice -> Expired / Cancelled -> Completed
```

- 活动项创建时立即消费本轮资格；重复 DeathEvent、第二次死亡、超时、旅行、重选、Pawn replacement、World Cleanup、PIE Stop 与 Deinitialize 均不得返还。
- 选择窗口固定 30 秒。超时保持 Dead，不应用 Delta、不请求保存。
- `E / Input.Interact` 选择 Rescue；`R / Input.QTE.Reject` 选择 Leave；`F / Input.Execute` 选择 FaceChallenge。不得新增 InputAction 或 GameplayTag。
- Rescue：`FPRReviveRequest` 固定 `HealthFraction=0.25`、`ShieldFraction=0`，FutureDisposition 为 `RescueEvacuationRequested`。
- Leave：不调用 Revive，保持 Dead，FutureDisposition 为 `LeaveRunRequested`。
- FaceChallenge：`HealthFraction=0.10`、`ShieldFraction=0`，FutureDisposition 为 `ChallengeContinues`。
- Rescue/FaceChallenge 只有 Revive 实际 Applied 后才请求关系 Delta；Revive 失败则不写关系、不保存。Leave 直接请求 Delta。
- 仅实际关系变化才调用一次 `RequestSaveCurrentProfile`。

固定关系 Delta（Trust/Affection/Evaluation/Overload）：

| Primary | Rescue | Leave | FaceChallenge |
|---|---:|---:|---:|
| Axiom | +2/+1/+1/+20 | -1/-1/+1/0 | 0/0/+2/0 |
| Kindle | +2/+2/0/+25 | -2/-1/0/0 | +2/+1/+2/0 |
| Null | +1/+2/+1/+20 | -1/-2/+1/0 | +1/+1/+2/0 |

固定主提示分别为：

- Axiom：“存续概率仍可修正。接受挽救、主动离开，或继续证明。”
- Kindle：“还没结束！让我拉你回来、现在撤，还是带着这点血继续冲？”
- Null：“删除流程卡住了。让我救、自己走，还是拿这点命继续赌？”

系统仅订阅值型 `FPRCombatEvent`、`FPRDialogueResult`、Primary Sync 和 Save Operation；Dialogue Result 只作为结果上下文，不读取 Dialogue 私有队列，也不改变资格或伤害。
