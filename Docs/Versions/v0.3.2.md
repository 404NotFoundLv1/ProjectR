# 目标

在单一完整版本中交付 Axiom、Kindle、Null 的十二个确定性 P0 QTE：DataAsset/Registry、唯一状态机、有限语义输入、只读 Prompt UI、关系结果与完整运行时清理。QTE 只订阅 `FPRCombatEvent`（含 AbilityOutcome）和 `FPRCompanionSupportEvent`；伤害、护盾、控制与无敌继续走既有 Combat/GAS 接口。

# 范围与冻结边界

- 不实现 Dialogue、LLM、画像、Director、Reward、Room、正式濒死保护、自动复活、Companion 新支援、敌人/Boss 新攻击、地图或 Save Schema 变更。
- 不修改 PlayerSkill、CombatSubsystem、AttributeSet、Enemy/Boss、HUD、既有 InputAction、AbilitySet、地图或 v0.2.4 平衡资产。
- 不创建 GameplayEffect、GameplayAbility、敌人、Boss、地图或第二套 UI 系统。
- QTE 不保存 Actor、ASC、Widget、Target、Timer、Delegate、RuntimeState、EffectHandle 或 SpecHandle。

# GameplayTag 与输入合同

只新增两个 Tag：

```text
Input.QTE.Reject
QTE.Result.Cancelled
```

最终为 129 explicit Tags、13 roots、63 checked getters。新增 `IA_QTE_Reject`，只映射 `R` 与 `Gamepad_DPad_Left`。`DA_PlayerInputConfig` 从 11 至 12 项，`IMC_Player` 从 26 至 28 项。普通 QTE 输入只复用现有 `Input.Attack`、`Input.Dodge`、`Input.Interact`、`Input.Execute` 和六项 `Input.Skill.*`；QTE 观察且提交语义输入，但不拦截、不替代 ASC Ability 路由。

# 公共值合同

`FPRQTERequest`：RequestId、QTEId、CompanionId、TriggerEventId、TriggerSource、SourceId、TargetId、AbilityTag、ResponseTags、WorldTimeSeconds。

`FPRQTERuntimeState`：RequestId、QTEId、CompanionId、QTEType、TargetId、State、AcceptedInputTags、StartTime、Deadline、RemainingSeconds。

`FPRQTEResult`：ResultId、RequestId、QTEId、CompanionId、ResultTag、TriggerEventId、TargetId、SubmittedInput、TimingGrade、AppliedEffectIds、RelationshipDeltaRequest、RelationshipApplyResult、SaveRequestId、ProfileSampleTags、WorldTimeSeconds。

所有结果都为值对象。未来版本只能订阅 `FPRQTEResult`，不得读取或控制 QTE 私有对象及 Timer。

# 状态机、去重与清理

```text
Idle -> Active -> Resolving -> Cooldown -> Idle
             -> Rejected / Failure / Timeout / Cancelled -> Idle
```

- 同时最多一个 Active 或 Resolving QTE；无队列、无陈旧请求复活。
- 基础全局冷却为 8 秒；实际冷却：`BaseCooldown * (1 + Clamp(Overload,0,100)/100)`。
- 资格：当前 Primary 匹配 CompanionId、Trust >= 40、Overload <= 90、玩家 Alive、Target 合法。
- 多候选顺序：Rescue > Defense > RuleCounter > Control > Attack；再按 DataAsset Priority、QTEId 字典序。
- CombatEvent 去重 64 条/2 秒；SupportEvent 去重 16 条/2 秒；请求去重 128 条/30 秒。
- 输入窗口前 35% 为 Perfect，仅写入 TimingGrade，不复制玩法效果。
- Death、Travel、Pawn replacement、World cleanup、PIE Stop、Deinitialize 均只发一次 Cancelled Result，不应用关系 Delta，不请求 Save。

# Registry 与 DataAsset

每个 `UPRQTEDataAsset` 固定拥有 PrimaryAssetId、CompanionId、QTEType、DisplayName、PromptText、AcceptedInputTags、WindowSeconds、PerfectWindowFraction、BaseCooldownSeconds、Trust/Overload 限制、Priority、严格 Trigger 条件、白名单效果定义，以及 Success/Failure/Rejected/Timeout Delta。

`DA_QTERegistry` 必须严格为 Axiom 四项、Kindle 四项、Null 四项，并持有 `WBP_QTEPrompt` SoftClass。Data Validation 拒绝未知 Tag、重复 QTEId、未知 Companion、非法效果、错误排序、非法窗口及非有限数值。Registry 只在世界初始化预加载；激活路径禁止同步加载。
