# v0.2.0 六个 P0 玩家技能

**状态**：In Progress

**直接上游**：v0.1.5

**直接下游**：v0.2.1

**未来消费者**：v0.2.1、v0.2.2、v0.2.3、v0.2.4、v0.3.2、v0.4.2、v0.4.4、v0.9.1、v1.0.1

## 目标

- 在 v0.1.3 冻结的 ASC、GameplayAbility、AbilitySet 和 InputTag 基础上实现六个可独立验证的 P0 玩家技能。
- 所有伤害只通过 v0.1.2 的 `UPRCombatSubsystem::ApplyDamage` 进入统一 Shield、Health、死亡和 CombatEvent 流程。
- 建立稳定、数据驱动的技能参数、目标选择、位移、防御、受击代理和非伤害结果事件合同。
- 复用正式输入语义并补齐键鼠/手柄映射，不建立数值 InputID 或第二套输入路由。
- 每个技能具备确定的阶段、Cost、Cooldown、失败路径、取消规则和安全占位表现。
- 将六技能拆为五个内部微任务；每个微任务都必须可独立 Build、测试和精确回滚，但外部版本始终保持 v0.2.0。
- 完成原生自动化、Blueprint 编译、精确保存、重启回载、PIE、完整历史回归、人工手感验收和 Future Compatibility Review。

## 本次范围

- 新增玩家技能公共类型、目标接口、技能 DataAsset、数据 Fragment、运行时 Component/Subsystem、公共 GameplayAbility 基类和六个原生技能类。
- 为统一伤害入口增量加入命中方向、命中原点、可阻断伤害和非伤害 AbilityOutcome 发布能力。
- 增加五个正式技能 InputTag，复用 `Input.Dodge`，并补齐五个 InputAction 与准确键鼠/手柄映射。
- 创建六个正式 Skill DataAsset、六个 GA Blueprint、十二个 Cost/Cooldown GE、四个状态 GE 和安全占位 VFX/SFX。
- 用六项 startup Entry 填充 `DA_DefaultAbilitySet`；授予、移除、死亡、复活和 Pawn 替换仍遵守 v0.1.3 合同。
- 实现影刃突刺、焚码斩、雷鸣落点、残影闪避、向量钩索和反证护壁的 P0 可玩闭环。
- 完成五个内部微任务、TDD、BuildEditor、原生自动化、MCP 资产生产、重启回载、PIE、历史回归和手感验收。

## Non-goals

- 不实现普通攻击；`Input.Attack` 与 J 只保持稳定语义，不授予 BasicAttack Ability。
- 不实现 v0.2.1 的敌人基类、小怪、精英或正式敌人业务。
- 不实现 v0.2.2 Boss、v0.2.3 HUD/技能栏/DamageNumber、v0.2.4 正式手感打磨或 v0.3.2 QTE。
- 不依赖具体敌人、Boss、QTE、Companion、Director、HUD、Widget、关卡或对话类型。
- 不实现技能升级、稀有度、奖励、构筑 Fragment、Save 持久化、输入重绑定 UI 或技能槽 UI。
- 不修改 Shield spill-over、Health Clamp、死亡、复活、默认属性一次应用或 CombatEvent 的既有伤害语义。
- 不替换 ASC、AttributeSet、PlayerState Owner/Character Avatar，不重命名现有属性、Tag、公共字段或资产路径。
- 不创建新的 Debug Console、任意 GA 类执行、任意资源修改或 Cooldown 清除入口。
- 不在 ProjectR 主模块 include ProjectRDebug；Debug 若未来接入技能，只能通过 Provider 和正式受控 Ability API。
- 不制作正式动画状态机、Montage、电影级 Niagara、最终 SFX、Camera Shake、手柄震动或 v0.2.4 平衡内容。
- 不增加永久 Tick；计时、阶段切换、持续效果和表现使用 GAS Duration/Period、Timer、Task、Delegate 或事件。
- 不增加新地图，不修改正式地图、GameMode、Save Schema、Config 中除 GameplayTag 外的内容或 `.uproject`。

## 公共版本与内部微任务规则

- `Docs/Roadmap/RoadmapErrata.md` 的 E-003 已冻结“UE 资产由 Codex + Unreal MCP 生产”，E-006 已冻结“六技能拆成五个内部微任务”；本版本沿用既有勘误，不新增重复路线勘误。
- v0.2.0-A 至 v0.2.0-E 是内部实现检查点，不是公共版本，不创建 Git Tag，不写入 VersionIndex，不单独推进 `CURRENT_VERSION.md`。
- A-E 必须按顺序完成；后续微任务只能建立在前一微任务 Build 与对应自动化 PASS 的状态上。
- 每个微任务必须维护准确 Allowed paths、UE Package 子 Manifest、MCP 操作记录和测试结果；失败只回滚该微任务的准确 diff。
- 任一微任务完成后 `CURRENT_VERSION.md` 仍为 v0.2.0/In Progress；只有 A-E、全部 required checks、人工手感和 Future Compatibility Review 全部 PASS 后，v0.2.0 才能标记 Completed。
- 内部微任务可使用本地检查点提交辅助回滚，但不得发布为独立版本；最终 v0.2.0 实施提交必须完整包含六技能并保持单一公共版本语义。

- `TWeakObjectPtr<AActor> TargetActor`

- `NaN`、Inf、负时长、负距离、无效 Actor 或跨越阻挡均返回失败，不产生 Cost、Cooldown、伤害或 Outcome。
- 位移使用受控 Movement/RootMotion/Timer 路径；不得 `SetActorLocation` 穿透阻挡或使用永久 Tick。

- `TSubclassOf<UGameplayEffect> CostEffectClass`
- `TSubclassOf<UGameplayEffect> CooldownEffectClass`

- `float RecoveryDuration`
- `float Range`
- `float Radius`

- `TSoftObjectPtr<UAnimMontage> Montage`

规则：

- DataAsset 是 P0 技能调参的唯一数据源；GA CDO、GE 和 DataAsset 中重复的 Cost/Cooldown/Tag 必须在 Data Validation 中逐项相等。
- Animation 引用允许为空；空引用只能触发安全占位反馈，不能阻止技能的客观逻辑验收。
- VFX/SFX 为安全占位内容，不代表 v0.2.4 最终美术与音频质量。
- SkillFragment 只承载当前技能确有需要的结构化扩展，不预建奖励、升级、QTE、敌人或 HUD 字段。

### `UPRPlayerSkillFragment`

- 原生基类为 `UObject`，使用 `EditInlineNew` / `DefaultToInstanced`，只允许作为 Skill DataAsset 的内联、受验证扩展对象。
- v0.2.0 基类没有未来业务字段；六技能仅在确有独立结构化参数且公共 DataAsset 字段无法表达时新增准确派生 Fragment。
- Fragment 不持有 ASC、SpecHandle、ActiveEffectHandle、Target、Timer、Widget 或 Save 状态，不执行伤害、位移或事件发布。

### `IPRAbilityTargetInterface`

```text
FName GetAbilityTargetId() const
FVector GetAbilityTargetPoint() const
EPRAbilityTargetMobility GetAbilityTargetMobility() const
bool CanBeTargetedByAbility(FGameplayTag AbilityTag) const
```

- Target selector 只依赖该接口、Actor、ASC 和 GameplayTag；不得 include 敌人、Boss 或 QTE 类型。
- 测试代理和未来 v0.2.1 敌人均实现同一接口。
- TargetId 必须稳定且非 `NAME_None`，用于确定性排序和自动化断言。

### `IPRCombatEventSubjectInterface`

```text
FName GetCombatEventSubjectId() const
```

- 为没有 ASC/Combatant 身份但需要出现在非伤害 AbilityOutcome 中的通用 Actor 提供稳定 SubjectId。
- `PublishAbilityOutcome` 的 TargetId 解析顺序固定为既有 `IPRCombatantInterface::GetCombatantId`，其次为该接口；两者都不可用时 Target 必须为空。
- 接口不暴露 HUD、QTE、敌人、Boss、对话或统计类型，也不允许消费者反向控制 Subject。

### `IPRCombatAttackProxyInterface`

```text
FName GetAttackProxyId() const
AActor* GetAttackProxyOwner() const
bool ConsumeAttackProxy(AActor* Attacker)
```

- 残影只提供一次性受击代理，不模拟正式敌人 AI、威胁表、仇恨、导航或伤害来源。
- `ConsumeAttackProxy` 必须幂等；首次成功后立即失效，重复调用无事件、无副作用。

### `IPRCombatMitigationInterface`

```text
EPRCombatMitigationResult EvaluateIncomingDamage(
    const FPRDamageRequest& Request,
    FGameplayTagContainer& OutResponseTags) const
```

- `UPRCombatSubsystem` 在 Authority 上调用；实现方只能返回是否完整阻断并追加 `Combat.Response.*` Tag。
- 实现方不得直接修改 Health/Shield、应用 GE、广播 CombatEvent 或决定死亡。
- `Blocked` 返回 `EPRCombatRequestStatus::RejectedBlocked`，属性不变，并广播一次 `Combat.Event.DamageRejected`。

### `UPRPlayerSkillComponent` 与 `UPRPlayerSkillSubsystem`

- `UPRPlayerSkillComponent` 由当前 `APRPlayerCharacter` 持有，只负责阶段状态、受控位移、状态/表现挂接和 Avatar 生命周期清理；不授予 Ability、不直接扣属性、不选择具体敌人。
- `UPRPlayerSkillSubsystem : UWorldSubsystem` 负责确定性目标查询、位移请求调度、世界级 Timer/代理清理和通用 AbilityOutcome 请求；不持有 HUD、QTE、敌人、Boss、Director 或 Save 状态。
- Character 只通过 Component/接口转发 Avatar 行为；ASC/PlayerState 继续持有 Spec 与 AbilitySet 生命周期。
- 两者均不得永久 Tick；世界清理、Avatar 替换和 EndPlay 必须使旧弱引用、Timer、位移和代理失效。

## GameplayAbility、AbilitySet 与 GAS 合同

- 六个正式 GA 均继承 `UPRPlayerSkillGameplayAbility`，继续使用 `InstancedPerActor`。
- Skill DataAsset 的 AbilityTag 在授予时进入 Spec 动态源 Tag；InputTag 只来自 AbilitySet Entry，不复制到 GA CDO。
- `DA_DefaultAbilitySet` 固定按以下顺序包含六个 startup Entry：影刃突刺、焚码斩、雷鸣落点、残影闪避、向量钩索、反证护壁。
- 每项 Level=1、`bGrantOnInitialization=true`、SourceObject 为对应 Skill DataAsset；Spec `InputID=-1`。
- PlayerState Authority 初始化只通过既有 `GrantAbilitySet` 幂等授予；Pawn 替换、死亡或复活不得重新授予或改变 GrantId/SpecHandle。
- 所有会产生伤害、位移、状态、残影或防御的 Ability 必须先通过目标/路径验证，再成功 `CommitAbility`，最后才产生业务后果。
- Cost GE：Instant、Energy Additive 负 Modifier、无 Execution/Period/Cue/Granted Ability/Stacking。
- Cooldown GE：Has Duration、零 Modifier/Execution、只授予对应唯一 `Cooldown.Skill.*` Tag。
- Cooldown 检查失败不得扣 Energy；Cost 检查失败不得创建 Cooldown；Commit 后半段失败遵守 v0.1.3 的准确 Cooldown 回滚合同。
- Dead 时所有六技能均被 `State.Dead` 阻断；现有死亡取消和 Revive 输入恢复合同不变。
- `State.Stunned` 阻断主动技能并取消正在执行的位移/引导；不会重复授予或清理 AbilitySet。
- 六技能不得直接调用 AttributeSet setter 扣血、改 Energy 或切换生命状态。

## InputTag 与物理输入合同

新增五个 InputTag，残影闪避复用既有 `Input.Dodge`：

| 技能 | AbilityTag | InputTag | 键盘 | 手柄 |
|---|---|---|---|---|
| 影刃突刺 | `Skill.ShadowThrust` | `Input.Skill.ShadowThrust` | U | Left Shoulder |
| 焚码斩 | `Skill.FireSlash` | `Input.Skill.FireSlash` | I | Right Trigger |
| 雷鸣落点 | `Skill.ThunderDrop` | `Input.Skill.ThunderDrop` | O | Left Trigger |
| 残影闪避 | `Skill.AfterimageDodge` | `Input.Dodge` | L / Left Shift | Face Button Right |
| 向量钩索 | `Skill.VectorHook` | `Input.Skill.VectorHook` | H | D-Pad Up |
| 反证护壁 | `Skill.CounterProofWall` | `Input.Skill.CounterProofWall` | Semicolon | D-Pad Down |

兼容映射：

- 保留现有 Left Mouse Button=`Input.Attack`、Space=Jump、Left Shift=`Input.Dodge`。
- 新增 J=`Input.Attack`、K=Jump、L=`Input.Dodge`；J 只验证语义输入，不实现普通攻击。
- A/D、左摇杆、Y 平面锁定、空中反向、Mesh 转向和固定相机合同不变；W/S 不新增 Y 平面移动。
- 现有 Face Button Right=`Input.Dodge` 保持不变，不重复添加映射。
- InputConfig 语义项由 6 增至 11；IMC 映射由 13 增至 26。
- 五个新 InputAction 只用于五个新技能；残影闪避继续复用 `IA_Dodge`。
- Character 继续只把 InputTag Pressed/Released 转发给 ASC，不 include 具体技能类。

## 目标选择合同

- Target selector 由 `UPRPlayerSkillSubsystem` 提供只读、确定性的 C++ 查询，不开放任意 Blueprint Class/Trace 参数入口。
- 候选必须：有效、非 Source、实现 `IPRAbilityTargetInterface`、允许对应 AbilityTag、具有有效目标点、未 `State.Dead`，且满足距离/角度/平面/LOS 条件。
- 排序固定为：到 Origin 距离升序 → 与 AimDirection 夹角升序 → TargetId 字典序；相同输入必须返回相同结果。
- Trace 只查询约定 Object/Collision 通道；WorldStatic 阻挡 LOS 与位移，Overlap 不得把未来 HUD、QTE 或无接口 Actor 当作目标。
- `ForwardSweep` 沿角色朝向和实际位移路径查询；同一激活对同一 TargetId 最多命中一次。
- `ForwardArea` 使用范围与半角；`GroundArea` 使用合法落点与半径；`SingleTarget` 选择唯一确定候选；`Self` 不查询外部目标。
- 没有目标、目标失效、路径受阻或位移非法必须在 Commit 前失败；失败 Tag 分别使用 `NoTarget`、`Obstructed`、`InvalidMovement`。
- v0.2.0 的原生自动化使用轻量测试代理；不得为了测试提前制作 v0.2.1 敌人。

## 伤害、CombatEvent 与防御合同

### 统一伤害公式

```text
RawDamage = (BaseDamage + AttackPower * AttackPowerScale)
            * (bCritical ? 1.5 : 1.0)
```

- AttackPower 在激活 Snapshot 时从同一 AttributeSet 读取；不得由 DataAsset、HUD 或 Debug 覆盖。
- 影刃突刺、焚码斩首击和雷鸣落点允许传播已确定的 Critical 标记；v0.2.0 不实现随机暴击判定。
- 焚烧 Tick 不可暴击。
- 每次实际伤害都构造一个 `FPRDamageRequest`：SourceId、DamageSource、Instigator、Target、AbilityTag、RawDamage、Critical、DamageTags、ImpactOrigin 和 IncomingDirection 必须准确。
- CombatSubsystem 继续负责最大值 Clamp、Shield spill-over、Health Clamp、死亡、CombatEvent 和反馈；技能不得复制这些职责。

`FPRDamageRequest` 增量新增：

- `FVector ImpactOrigin`
- `FVector IncomingDirection`

规则：

- 两者必须 finite；IncomingDirection 必须能安全归一化。环境伤害可使用零方向，玩家技能不得使用零方向。
- 既有调用方采用向后兼容默认值；不得重命名或删除 v0.1.2 字段。

`EPRCombatRequestStatus` 增量新增：

- `RejectedBlocked`

伤害处理顺序冻结为：

1. 结构、数值、Target、Authority 校验。
2. Dead 拒绝。
3. `IPRCombatMitigationInterface` 防御评估。
4. Invulnerable 拒绝。
5. GAS Execution、Shield/Health 结算、死亡和事件。

防御发生在 Invulnerable 之前，以保证正面护壁命中稳定记为 GuardBlocked；防御与无敌均不得重复扣血或广播 Death。

### 非伤害 AbilityOutcome

`FPRCombatOutcomeRequest` 固定包含：

- `FName SourceId`
- `TWeakObjectPtr<UObject> AbilitySource`
- `TWeakObjectPtr<AActor> Instigator`
- `TWeakObjectPtr<AActor> Target`
- `FGameplayTag AbilityTag`
- `FGameplayTagContainer ResponseTags`

`UPRCombatSubsystem` 增量提供：

```text
bool PublishAbilityOutcome(const FPRCombatOutcomeRequest& Request)
```

- 只允许 Authority、有效 SourceId/AbilityTag/Instigator 和非空 ResponseTags。
- 使用既有 `FPRCombatEvent` 发布 `Combat.Event.AbilityOutcome`；伤害数值字段固定为 0，`bFatal=false`。
- 仅用于位移完成、残影创建/消耗、护壁阻断与完美时机等稳定战斗事实；不用于 Ability 生命周期、UI 动画、QTE 结果或 Debug 日志。
- Combat 仍不得 include 具体技能、敌人、Boss、HUD、QTE、Companion 或 Director 类型。

## 六技能冻结行为

### 影刃突刺

- Input：U / Left Shoulder；策略 `OnInputTriggered`。
- Energy Cost=20；Cooldown=4.0s。
- 阶段：Startup 0.08s → 450cm/0.18s 前突 Active → Recovery 0.12s。
- 目标：`ForwardSweep`，路径半径 75cm；WorldStatic 阻挡时在阻挡点前停止。
- 每个候选 TargetId 在一次激活中最多命中一次。
- 伤害：Base 25、AttackPowerScale 1.5，允许传播 Critical。
- Commit 前路径或位移非法返回 `Ability.ActivateFail.InvalidMovement`；Commit 后死亡、Stunned、Avatar 替换或阻挡中断取消剩余位移，不返还 Cost、不清除 Cooldown、不补发伤害。
- 成功位移追加 `Combat.Response.DisplacementApplied` AbilityOutcome。

### 焚码斩

- Input：I / Right Trigger；策略 `OnInputTriggered`。
- Energy Cost=25；Cooldown=5.0s。
- 阶段：Startup 0.15s → 前方 360cm、120 度扇形 Active → Recovery 0.20s。
- 首击伤害：Base 15、AttackPowerScale 1.0，允许传播 Critical。
- 命中目标获得 `State.Burning`，0.5s 间隔执行 3 次 Tick；每 Tick Base 4、AttackPowerScale 0.1，不可暴击。
- 同一来源再次命中只刷新持续时间和 Tick 计数，不叠加第二个 Burning 实例；不同来源在 v0.2.0 也不叠加。
- 目标在 Tick 前 Dead、失效或 Avatar/Owner 不再匹配时停止；Tick 仍逐次走统一伤害入口。
- Startup 取消不 Commit；Commit 后取消不撤销已经成功应用的 Burning GE。

### 雷鸣落点

- Input：O / Left Trigger；策略 `OnInputTriggered`。
- Energy Cost=30；Cooldown=7.0s。
- 目标：优先选择 700cm 内合法地面目标；无合法目标时使用前方 450cm 的可达落点。
- 阶段：确认落点并 Commit → 0.60s 延迟 → 半径 220cm GroundArea 命中。
- 伤害：Base 25、AttackPowerScale 2.0，允许传播 Critical。
- 存活目标获得 `State.Stunned` 0.75s；Stunned 只通过准确 Duration GE 管理。
- 落点非法、被 WorldStatic 遮挡或无法投射到允许平面时在 Commit 前失败并使用 `Obstructed`/`NoTarget`。
- Commit 后 Source 死亡不取消已经生成的落点；世界清理或 Subsystem 销毁时安全取消 Timer，不访问失效对象。

### 残影闪避

- Input：L / Left Shift / Face Button Right；策略 `OnInputTriggered`。
- Energy Cost=15；Cooldown=2.5s。
- 阶段：立即 Commit → 300cm/0.18s 位移 → Recovery 结束。
- `State.Invulnerable` 持续 0.22s；完美时机窗口为开始后的 0.14s。
- 位移沿当前移动输入方向；无输入时沿角色朝向；必须保持 2.5D 平面并 Sweep 阻挡。
- 激活时在起点创建一个残影代理，最大存在 1.5s；首次合法敌对攻击消费后发布 `DecoyConsumed`，超时无额外事件。
- 创建发布 `DecoyCreated`，成功位移发布 `DisplacementApplied`；完美窗口内消费追加 `PerfectTiming`。
- 残影无 ASC、Health、AI、导航、碰撞伤害或正式敌人类型依赖；销毁/消费必须幂等。
- 位移非法在 Commit 前失败；Commit 后死亡、Stunned、Avatar 替换或阻挡取消剩余位移，但已产生的 Cost/Cooldown/短暂无敌不回滚。

### 向量钩索

- Input：H / D-Pad Up；策略 `OnInputTriggered`。
- Energy Cost=15；Cooldown=4.0s；伤害固定为 0。
- 目标：面对方向 800cm、45 度半角内唯一合法 `SingleTarget`，必须有 LOS。
- `Light`：目标在 0.30s 内被拉向玩家并停在 120cm；`Heavy`/`Anchored`：玩家被拉向目标并停在 120cm。
- 位移全程 Sweep，阻挡立即停止；不能穿过 WorldStatic、离开 2.5D 平面或移动 Anchored 目标。
- 没有目标返回 `NoTarget`；LOS/路径阻挡返回 `Obstructed`；两者都必须发生在 Commit 前，因此不扣 Energy、不创建 Cooldown。
- 成功完成发布 `DisplacementApplied` AbilityOutcome；中断时不发布完成事件。
- Commit 后目标失效、死亡、Stunned、Avatar 替换或阻挡取消位移，不返还 Cost/Cooldown。

### 反证护壁

- Input：Semicolon / D-Pad Down；策略 `WhileInputActive`。
- Energy Cost=20；Cooldown=6.0s；伤害固定为 0。
- Press 上升沿验证、Commit 并进入 `State.Guarding`；最多持续 1.0s，Release 提前结束。
- 正面判定为玩家朝向中心的 60 度半角；IncomingDirection 为零、非 finite 或来自背面时不得阻断。
- 正面合法伤害被完整阻断，属性不变，响应包含 `Combat.Response.GuardBlocked`。
- 激活后前 0.15s 为完美时机；该窗口内阻断额外包含 `Combat.Response.PerfectTiming`。
- 一次激活可阻断多次合法正面伤害，但不反击、不伤害、不刷新 Duration、不刷新 Cooldown、不返还 Energy。
- Release、1.0s 到期、死亡、Stunned、Avatar 替换或 AbilitySet 移除都取消 Guarding；重复 Release/取消无副作用。
