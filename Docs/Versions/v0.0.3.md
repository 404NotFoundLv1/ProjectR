# v0.0.3 创建核心地图与 Game Framework
**状态**：In Progress

**目标**：建立主菜单、现实据点、网络关卡、测试场的基本流转。

## 前置状态
- v0.0.2 已完成。

## 本次范围
- 创建核心类：GameInstance、GameModeBase、RealityGameMode、NetworkRunGameMode、PlayerController、PlayerState、PlayerCharacter。
- 创建地图：L_MainMenu、L_RealityHub、L_Network_Prototype、L_CombatGym、L_BossGym。
- 设置地图默认 GameMode。
- 创建基础 OpenLevel 流程。

## Non-goals
- 不实现下一公共版本。
- 不做无关重构。
- 不静默破坏已记录接口、Tag、Save 字段或资产路径。

## Allowed paths
- `Source/ProjectR/Core/**`
- `Source/ProjectR/Characters/**`
- `Config/DefaultEngine.ini`
- `Content/ProjectR/Maps/**`
- `Content/ProjectR/Blueprints/**`
- `Docs/**`

## Forbidden paths/actions
- Deleting template Variant
- GAS/combat/companion/director business logic
- unapproved asset moves

## 跨版本关系
- Depends on: v0.0.2。
- Produces: 建立主菜单、现实据点、网络关卡、测试场的基本流转。
- Consumed by: v0.0.4。
- Extension points: 配置、接口骨架、验证脚本、MCP Authoring 入口。
- Must remain compatible: 工程名、模块名、目录、构建入口、GameplayTag 命名。

## 本版本冻结合同
- GameInstance/GameMode/PlayerController/PlayerState/PlayerCharacter lifecycle
- formal map IDs and travel entry

## Codex C++/文件工作
- 先检查真实状态和依赖，再给准确 diff 计划。
- 使用 apply_patch 修改文本文件。
- 执行适用 Build/静态校验并报告实际结果。
- 不把未运行写成通过。

## MCP Operation Manifest
- Duplicate approved template map, create thin Blueprint subclasses, set parents/CDO/GameMode, place spawn actors, save/compile/load/PIE.
- 写入前列出准确 Package/Tool/顺序/停止点；写入后分别报告保存、编译、重启加载和 PIE。

## Human Judgment / Credential
- Judge camera composition and 2.5D framing; provide PASS/FAIL and screenshots.
- 若需要人工，必须引用 ManualOperationsRunbook 的精确步骤；用户不负责手动编码或重复资产配置。

## 自动验证
- 逐张地图 PIE。
- OpenLevel 测试。
- 检查保护路径、公共合同、KnownIssues 和 Future Compatibility Review。

## 验收标准
- 所有地图可打开。
- 玩家在每张测试地图正常出生。
- 能从主菜单进入现实据点。
- 下游版本能通过记录的扩展点继续实现，无需大规模改写本版本。

## 回滚条件
- Build/Editor/PIE 出现当前版本引入的阻断。
- 需要未批准的保护路径或破坏性资产操作。
- 当前实现要求提前创建未来系统。
- Blueprint/Save/Tag/资产路径出现未记录兼容破坏。
